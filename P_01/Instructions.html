Assignment Description:

You are required to create the interfaces for a small e-commerce web application. The application will have the following interfaces:

  1. Home Page with links
  2. Login form
  3. Registration form
  4. Product List page
  5. Shopping Cart page
  6. Order list page

You have to customize the interfaces with your own images and interest topics.

Remember that you'll need to RESEARCH things independently during practice, as demonstrated by the instructor. If, at any point, after attempting to solve it alone, you can't find a solution, you can consult your instructor for assistance.

To start the practice, create a folder named P_01. Everything you do should be within this folder, as this is what you'll need to upload to the platform as a zip file.

1. Home Page

This is the landing page for the application that looks like:

  1. Create an html file named P01_index.html with Bootstrap 5 (bs5-$). You can use the Bootstrap5 snippets extension. You can leave the structure as it is with the template that is generated automatically.
  2. Use a navbar and a carousel to create the first part of the interface. Place three images in the carousel. THE NAVBAR MUST BE PLACED ON ALL PAGES OF THE PRACTICE. Consider how the search bar is arranged and the text it contains. Also, consider that there is a logo. Clicking on this logo should take you to the home page.
  3. For the user account and shopping cart icons, use FontAwesome or Bootstrap icons
  4. For the products section, use Bootstrap 5 cards.
  5. Ensure that the images have a defined size to display well on the screen (you can use max-height and min-height), and distribute the product cards according to the screen size logic: for large screens, display 4 columns, for medium screens, 3 columns, and for small screens, two columns. For this, you can use the Bootstrap 5 template bs5-modal-default and work on that template.
  6. To verify that this works, you must include at least 8 products.
  7. For images, you can use websites such as: Free Images. You can copy and paste the image URL and set it as the src, so you don't have to download and arrange the image.
  8. At the end, add a pagination component using another snippet bs5-modal-default. You can center the navbar with the class justify-content-center.

2. Login Form (modal)

Clicking on the "Login" navbar should open a modal window like the one shown:

  • You can define your modal within the <head>.
  • Create a modal using the snippets extension (bs5-modal-default). It will generate it with a button and a small script that you can remove. YOU MUST REMOVE THIS BUTTON, and make the modal launch when clicking on the "Login" option in the navbar menu. To do this, change the data-bs-target of your "Login" link in the navbar to the ID you will assign to the modal. Pay attention to the ID because you will have to assign a different one in another modal that we will create later.
  • Ensure that, in the <a> tag of the navbar that will open your modal, you add role="button" so that the mouse cursor changes to a hand icon when hovering over it.
  • Both fields in the form must be within a <form>, and the "Login" button must have type="submit".
  • For each field, use divs with the classes input-group and mb-3, to display the icon next to it. You should have a span and an input inside these divs for the field and the icon. Check this out here: Bootstrap Input Groups
  • Add the required property to each input to validate that the fields have content when clicking on "Login".
  • The labels for "Email" and "Password" should reference their respective fields. Note that these labels should be outside the input-group, but they must have a for attribute corresponding to the id of the inputs.
  • Center the "Login" button by placing it inside a div and adding the style text-align: center;.
  • Add a checkbox with the text "Remember", to simulate keeping the session logged in.
  • The text "Don't have an account? Sign up here" should be in the modal-footer. The text "Sign up here" should open a modal for the next part of the practice.

3. Registration Form (modal)

Create another modal for user registration. Do not forget to change the ID of this modal, it must not be the same as the login modal. It should be as follows:

  • To have the first and last names appear on the same line, they must be within the same .row and each within a div with .col-6, inside a Bootstrap grid container. Just like with the products, you can use the bs5 template.
  • The fields SHOULD NOT have labels; they should have the placeholder shown in the image.
  • It is also not necessary for the register button to validate the fields.

4. Product List Page

Create a new file named P01_cart.html. You should be directed to this page when clicking on the shopping cart icon at the far right of your navbar. Remember that the header should be the same on all pages. It should be as follows:

  • For the product cards, you can use a media object. Even though the documentation is for BS4, you can use this reference link to know how to use it: Bootstrap Media Objects, or this Media Objects
  • Remember that, for the image, you can add the style float: right to make it appear on the right.
  • Inside the media-body class, you should put the fields shown in the image. For the fields, you can use another input-group with elements to the left and right of the field (for the "MXN" case).
  • The Delete button is a button inside the H4 that contains the product name.

5. Link everything

  • Make sure that from the home page, all links are functional; the navbar should take you to login and shopping cart.

  • For the "About Us" page in the navbar, simply create a P01_about_us.html page that contains the navbar and a simple H1.

  • For the product pages "Category A" and "Category B", create two HTML files named P01_cat_A.html and P01_cat_B.html. On these pages, you can copy and paste your product listing from the index to display 5 products on each page, with the paginator at the bottom. As shown in the following image:

  • Remember that all pages must have the navbar properly linked between them.

  • Don't worry about the search bar. For this practice, it just needs to be there, but it doesn't need to do anything in particular.

6. Responsive Design

Visualize your app in different window sizes and make it responsive using the classes .col-prefix(sm,md,lg,xl)-1,2,...12. You should have at least 2 versions: one for small screen devices and another for medium or larger sizes. This should also imply that the shopping cart page resizes in a way that you consider appropriate for the sizes you want to define. I will review that the Home page is responsive and that the shopping cart is responsive. TIP: Change the number of columns on the Home page based on the screen size, and change the layout of the shopping cart elements so that the totals appear below on small screens.

7. Order Details (Optional)

If you still have time, create a file "P01_orders.html" that displays the orders page with all the data of the purchased products. The format is free, but you should use images and bootstrap components like collapse, accordion, tables, etc.

In this practice, we were able to use forms and input validation, navigation links, Bootstrap layout, modal windows usage, adding specific styles with CSS, etc. We are ready to learn JavaScript to handle events and make our application more interactive.
